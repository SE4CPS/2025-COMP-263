1. Summary

This lab implements a browser-based agriculture data dashboard using IndexedDB. The app can insert up to 10,000 records containing: sensor readings, crop photos, farmer notes, GPS coordinates, and timestamps. The first 10 records can be displayed in the browser. Unit tests using console.assert() validate correctness and constraints.

2. Files Included

  • index.html — main webpage with UI and buttons
  • script.js — IndexedDB logic, insert/display, and tests
  • lab1-shaikh-0432.html — this explanation file
  • images/ — crop photos and placeholder image

3. How to Run

  1. Open index.html in a modern browser, or run a local server: 
    python -m http.server 8000
# Visit http://localhost:8000/labs/lab1/index.html
  2. Use the buttons to insert records, display data, run tests, or reset the DB.
  3. Check DevTools → Console for test results and logs.

4. Database Schema

Database: AgricultureDB
Object store: FarmData (auto-increment id)
Fields:
- sensorReadings: Array<number>
- cropPhoto: string (URL to local image)
- farmerNote: string
- gpsCoordinates: { lat: number, lon: number }
- timestamp: ISO string

5. Key Code Features

  • Batch insert up to 10,000 records
  • Display first 10 records with images
  • Unit tests for add/get, image extension, numeric arrays, GPS ranges, counts
  • Reset function to clear the DB
